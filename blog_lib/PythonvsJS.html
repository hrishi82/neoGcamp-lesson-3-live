My JS Journey

August, 2021

I learned python during my final year of M.Tech and the journey has been amazing so far. The idea to learn python came to me when I started applying for jobs. While applying, I gradually realized that I had to stand out from the crowd, so I learned python since it is the easiest programming language to learn.

2 years down the line, on the month of August this year I met JavaScript and its friends (HTML & CSS). I knew I had to graduate from the python class and since I fell in love with staring at 2 monitors while developing python selenium scripts, web development seemed like the perfect next chapter in my journey book. I feel that, had I known java or C++ earlier, JavaScript would have come easier to me but whatever. The journey has been both rocky and pleasant and I plan to continue this journey.

  • Callback functions:

    Soooooo many type of functions, I know! Callback functions put simply, are functions which are passed as parameters to another functions.

    We came across this during the 4th lecture of NeoG Camp Level 0. I did read up on this type of functions and came to know that there is a thing called asynchronous programming. What it means is that, suppose there is a line of code (we will keep it at line no. 16) and it is flanked by two other lines which are line 15 and 17. Now, upon execution, it is not necessary for line no 16 to execute after the code from line 15 has executed. The code from line 16 will run after some event is triggered or something happens.

    In the image displayed above, we can observe that inside the addEventListener method, after "click", we have dataFetch which is actually a function which is defined above but it is without a parenthesis which makes it a callback function!

    So, to call a callback function, we don’t use parenthesis. Just the function name would do. A good example of what this function does and how it works can be found HERE

Apart from these topics, these are some of my other observations:

  1. We can access global variables even if they are not passed explicitly to a function.
  2. "==" evaluates only the variables but not their dataType but "===" evaluated both the variables and their dataType.
  3. DONOT forget to check for ; while searching for errors!
